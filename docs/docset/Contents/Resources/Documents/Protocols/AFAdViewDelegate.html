AFAdViewDelegate Protocol Reference

Conforms to NSObject
Declared in AFAdViewDelegate.h

Overview

During the Fusion ad load and usage, the AFAdView can notify object conforming to the AFAdViewDelegate protocol about various events triggered during ad load or during user interaction with the ad.

The ad view behavior can also be controlled by returing specific values from some of the notification methods.

Notification called before media will be started (called through mraid.playVideo()), or when inline video starts and the creative implementation calls Fusion.SDK.inlineMediaWillStart.

- (BOOL)adView:(AFAdView *)adView willStartMediaWithUrl:(NSString *)url inline:(BOOL)isInline

Parameters

adView

The AFAdView that received request to play video.

url

The video URL.

isInline

Indicates whether the event was fired by the mraid.playVideo method (value is NO), or if it was called directly by the creative (value is YES)

Return Value

YES if the client wants to allow opening the video player and play the video; NO otherwise.

WARNING: returning NO has no effect on inline videos (will play regardless of returned value).

Implement this method if you want to override the detected calendar support of the device.
